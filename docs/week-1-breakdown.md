# 第一周任务拆解

时间范围：2026-03-23 至 2026-03-29

## 周目标

完成一个可运行的后端基础骨架，为后续 RAG 和 Agent Workflow 铺路。

本周结束时，至少要具备：

- FastAPI 服务可启动
- `health` 接口可用
- 配置管理可用
- 结构化日志可用
- 模型调用封装有统一入口
- 2 个工具具备最小可调用能力

## Day 1: 仓库初始化

交付目标：

- 明确目录结构
- 完成 `README.md`
- 输出技术方案草稿

具体任务：

- 创建 `app/`、`docs/`、`tests/`、`scripts/`
- 明确各目录职责
- 写清 MVP 边界和第一阶段目标

验收标准：

- 新人打开仓库后能在 1 分钟内理解项目做什么

## Day 2: FastAPI 骨架

交付目标：

- 完成基础应用入口
- 提供 `GET /health`

具体任务：

- 创建 `app/main.py`
- 初始化 FastAPI app
- 增加基础路由模块
- 约定统一返回结构

验收标准：

- 本地启动成功
- `GET /health` 返回正常

## Day 3: 配置与环境变量

交付目标：

- 完成配置分层
- 明确 `.env` 读取方式

具体任务：

- 设计 `Settings` 结构
- 约定环境变量命名
- 区分本地、测试、生产基础配置

验收标准：

- 能通过环境变量切换模型、日志级别、数据库配置

## Day 4: 日志与异常处理

交付目标：

- 实现结构化日志
- 实现统一异常处理

具体任务：

- 增加 request id
- 输出请求耗时
- 定义业务异常与系统异常
- 统一错误响应格式

验收标准：

- 错误场景能看到统一日志和清晰错误码

## Day 5: 模型层抽象

交付目标：

- 建立统一的 LLM 调用接口

具体任务：

- 设计 `ModelClient` 抽象
- 统一消息输入输出结构
- 预留 OpenAI 兼容接口适配层

验收标准：

- 业务层不直接依赖具体模型 SDK

## Day 6: Tool Use 最小闭环

交付目标：

- 接入网页搜索和网页抓取两个工具

具体任务：

- 定义工具输入输出 Schema
- 实现工具超时与失败回退
- 统一工具调用日志

验收标准：

- 能单独调用工具并拿到结构化结果

## Day 7: 周总结与整理

交付目标：

- 清理骨架代码
- 补充文档
- 准备下周进入 RAG

具体任务：

- 完善 README
- 整理待办与技术债
- 录制或编写一个最小 demo 流程

验收标准：

- 仓库具备继续开发的稳定基础

## 本周优先级

P0：

- FastAPI 启动
- 配置加载
- 日志与异常
- 模型抽象

P1：

- 搜索工具
- 网页抓取工具
- 统一返回结构

P2：

- Swagger 优化
- 示例请求
- 第一版测试

## 风险与控制

- 风险：过早接入太多框架，导致骨架不稳定
- 控制：第一周只做主链路，不做前端、不做复杂记忆

- 风险：工具调用和模型调用耦合过深
- 控制：先定义抽象接口，再接 SDK

- 风险：目录先搭太大，后续空壳过多
- 控制：保留目录，但只实现第一周实际用到的模块
